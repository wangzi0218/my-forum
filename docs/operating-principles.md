# 运行原则

这份文档记录新项目的产品和工程底线。

## 1. 不把聊天当全部

用户看到的是自然对话，但系统内部必须有工作区、案例、记忆和事件。

没有状态，就没有持续协作。

## 2. 不静默记忆

系统可以建议记住一条背景，但不能悄悄写入长期记忆。

长期记忆至少要让用户知道：

- 记了什么
- 记到哪里
- 为什么值得记
- 如何忽略或撤回

## 3. 不无脑顺着用户做方案

当用户直接说“做个弹窗”“加个入口”“生成方案”时，系统应该先判断问题是否成立。

如果证据不够，要敢于停下来。

## 4. 不追求一次问全

系统每轮只追最影响推进的一两个问题。

产品经理不是在填问卷。

## 5. 不把 LLM 当运行时

LLM 可以理解语义、整理表达、归一化草稿。

但这些事情不应该交给 LLM：

- 是否写入长期记忆
- 是否越过阶段关口
- 是否执行高风险动作
- 是否覆盖稳定项目背景

## 6. 不急着做完整平台

第一版先证明一个真实闭环。

不要一开始就做：

- 登录注册
- 团队权限
- 复杂后台
- 多 agent 编排
- 大量集成入口

## 7. 每次输出都要让用户知道下一步

一张好卡片至少回答：

- 我现在怎么理解
- 我卡在哪里
- 下一步最值得补什么
- 有没有必要先停

如果输出只是“看起来有道理”，那就没有帮用户推进工作。
