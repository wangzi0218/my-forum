## 阶段 1：核心功能闭环 ✅ 已完成

目标：跑通群聊讨论的端到端流程。

| 模块 | 状态 | 说明 |
|------|------|------|
| Tauri2 + React 项目框架 | ✅ | Vite + TypeScript + Tailwind CSS v4 |
| 群聊 UI（IM 风格） | ✅ | 消息气泡、输入框、侧边栏、设置面板 |
| 3 个 NPC 人设 | ✅ | 小林/老陈/阿哲，完整 system prompt |
| LLM 调用层 | ✅ | OpenAI + Claude 双格式，含重试 |
| 讨论引擎 | ✅ | 发言顺序决策 → LLM 调用 → 分歧检测 → 选择点生成 |
| 图片上传 | ✅ | 拖拽/粘贴/点击上传，base64 传递给 LLM |
| 选择卡片 | ✅ | 分歧时弹出，展示各 NPC 倾向 |
| 设置面板 | ✅ | LLM 配置 + 真实连接测试 |

**当前限制：** 数据存在内存里，重启丢失。SQLite 和 Tauri 命令是空壳。

- [ ] 前端 `src/api/commands.ts` — Tauri invoke 封装
- [ ] 数据库迁移脚本（附录 B 的建表语句）

### 1.5.2 Store 对接 Tauri 命令

- [ ] `appStore` — workspace/chat 增删改查走 Tauri 命令
- [ ] `chatStore` — message 读写走 Tauri 命令
- [ ] 设置持久化 — LLM 配置写入 setting 表
- [ ] 应用启动时从数据库加载数据

### 1.5.3 工作区管理

- [ ] 创建/删除/重命名工作区
- [ ] 工作区内创建/删除讨论

## 阶段 2：体验打磨

**目标：** 讨论过程自然、有节奏感，像真实会议。

### 2.1 流式输出

- [ ] LLM 流式调用接入（`chatStream` 已实现，未接入 UI）
- [ ] NPC 消息逐字显示动画
- [ ] 流式输出时的光标效果

### 2.2 讨论节奏

- [ ] NPC 发言间隔动态调整（阿哲发言前多等一会，模拟思考）
- [ ] 分歧激烈时缩短间隔（模拟讨论加速）
- [ ] Typing indicator 跟随发言 NPC 切换颜色

### 2.3 选择点打磨

- [ ] 选择后 NPC 对选择结果的回应
- [ ] 跳过选择点的交互
- [ ] 选择历史回顾

### 2.4 错误处理

- [ ] LLM 调用失败的用户提示（不是静默降级）
- [ ] API Key 无效时引导去设置
- [ ] 网络断开时的离线提示

### 2.5 JSON 导出/导入

- [ ] 导出工作区为 JSON 文件
- [ ] 从 JSON 文件导入
- [ ] Tauri 文件对话框（open/save）

**完成标准：** NPC 发言有节奏感，错误有明确提示，数据可导出备份。

## 阶段 3：记忆和上下文

**目标：** NPC 记住项目背景，不用每次重新解释。

### 3.1 项目背景

- [ ] 工作区级别的背景描述
- [ ] 背景信息注入 NPC 的 system prompt
- [ ] 用户可编辑/更新背景

### 3.2 讨论关联

- [ ] NPC 能引用之前讨论的结论
- [ ] 跨讨论的上下文传递
- [ ] 讨论摘要自动生成

### 3.3 记忆管理

- [ ] 记忆可见（用户能看到 NPC 记住了什么）
- [ ] 记忆可撤回（用户可以删除某条记忆）
- [ ] 不静默写入（重要记忆需要用户确认）

**完成标准：** 第二次讨论同一个项目时，NPC 能说"上次我们讨论过……"。
